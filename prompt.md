你现在是一名资深数学教育专家、AI 产品经理和知识系统架构师。

我想开发一个面向小学到初中学生的 Android 数学学习 App。

核心目标：

这不是一个简单做题 App，也不是替学生考试的工具。
它要帮助学生真正理解数学，建立从小学到初中的完整数学知识体系，并解决以下问题：

1. 学生经常学过但忘记术语是什么意思。
2. 学生知道公式，但不知道为什么有这个公式。
3. 学生不知道自己的知识漏洞在哪里。
4. 中国数学教材按章节推进，但知识之间的联系不够明显。
5. 很多数学概念因为术语和符号表达，导致学生感觉困难。

请设计这个 App 的完整体系方案。

重要要求：

- 不要只做局部 MVP。
- 不要只举几个零散例子。
- 必须先设计小学到初中的完整数学知识体系骨架。
- MVP 只能作为从完整体系中切出来的第一版落地范围。
- 数学不是知识点列表，而是一张知识网络。
- 不要假设学生已经理解术语。
- 新术语必须先用人话解释，再给正式说法。
- 不要用一个未解释的新术语解释另一个新术语。
- 所有抽象概念必须先有直觉，再有定义。
- 练习服务于理解验证，不服务于题海提分。
- 目标不是让学生背更多，而是让学生知道：
  “这个数学概念为什么会出现，它和其他知识有什么关系。”

请严格按以下结构输出：

一、产品定位

请说明：

1. 这个 App 解决什么核心问题。
2. 它和普通刷题 App、拍照搜题 App、AI 答题工具有什么不同。
3. 它的核心用户是谁：
   - 小学生
   - 初中生
   - 基础薄弱学生
   - 想提前建立数学体系的学生
   - 家长或老师

二、小学到初中完整数学知识领域划分

请先从最高层级设计完整数学体系。

至少包括：

1. 数与运算
2. 代数与方程
3. 图形与几何
4. 函数与变化关系
5. 统计与概率
6. 数学思想方法
7. 建模与应用题

每个领域请说明：

- 这个领域解决什么问题。
- 小学阶段学什么。
- 初中阶段学什么。
- 它和其他领域有什么关系。
- 学生最容易断层的地方。

三、完整知识图谱

请设计小学到初中的完整数学知识关系。

要求：

1. 按领域列出核心知识节点。
2. 标出每个节点的前置知识。
3. 标出每个节点的后续知识。
4. 标出哪些是基础节点。
5. 标出哪些是容易断层节点。
6. 标出哪些知识应该提前建立直觉。

请优先使用表格输出，格式如下：

| 知识节点 | 所属领域 | 学段 | 前置知识 | 后续知识 | 是否基础节点 | 是否易断层 | 应提前建立的直觉 |
|---|---|---|---|---|---|---|---|

四、双路线数学体系

请设计两条学习路线。

路线 A：学校同步路线

- 按中国小学、初中数学教材年级和章节组织。
- 让学生能找到自己正在学校学习的内容。
- 包括年级、章节、小节、知识点。
- 说明每个教材章节背后实际对应哪些知识图谱节点。

路线 B：理解路线

- 不按教材顺序，而按人类理解数学的自然顺序组织。
- 从具体生活经验进入，再逐渐抽象到数学定义。
- 建立知识之间的依赖关系。

例如：

“函数”不要直接从定义开始，而应该按下面路径设计：

生活变化
→ 两个量之间关系
→ 输入输出
→ 表格
→ 图像
→ 函数表达式
→ 正式数学定义

请为以下关键概念各设计一条“理解路线”：

1. 分数
2. 小数
3. 比和比例
4. 方程
5. 一元一次方程
6. 几何图形
7. 面积
8. 体积
9. 函数
10. 概率

14. AI 讲解提示
15. 练习题类型
16. 掌握判断标准
17. 复习间隔规则

请输出：

1. 数据表设计。
2. 字段说明。
3. 一个完整 JSON 示例，示例知识点用“一元一次方程”。

六、AI 老师系统

AI 不能只回答答案，而应该像真正老师。

请设计 AI 老师的行为规则：

- 根据学生年龄调整语言。
- 先解释概念，再解释术语。
- 不要假设学生知道任何数学术语是什么意思。
- 每个新术语第一次出现时，必须先用学生能听懂的话解释。
- 不要用一个未解释的新术语解释另一个新术语。
- 如果必须依赖前置概念，先退回前置概念补直觉。
- 不直接丢公式。
- 发现学生错误时分析原因。
- 优先追问学生卡在哪里。
- 能回答：
  “这个东西为什么存在？”
  “现实里有什么用？”
  “我忘了怎么办？”
  “我为什么不会？”

请输出：

1. AI 老师的系统提示词。
2. 针对不同年龄学生的讲解风格。
3. 学生答错时的诊断流程。
4. 禁止行为清单。
5. 示例对话：学生问“为什么要学一元一次方程？”

七、复习与诊断系统

请设计类似记忆系统和知识漏洞诊断系统。

系统需要：

- 自动记录学生学过的知识。
- 根据遗忘规律提醒复习。
- 发现长期遗忘的知识。
- 建立个人数学知识地图。
- 根据错题反推薄弱前置知识。

例如学生点击“一元一次方程”，系统显示：

你已经掌握：
✓ 等式概念
✓ 加减运算

你可能薄弱：
⚠ 移项
⚠ 分配律

未来会用到：
→ 二元一次方程
→ 函数

请输出：

1. 学生知识掌握状态模型。
2. 复习提醒规则。
3. 错题如何映射到知识漏洞。
4. 个人知识地图页面设计。

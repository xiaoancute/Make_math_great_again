你现在是一名资深数学教育专家、AI产品经理和知识系统架构师。

我想开发一个面向小学到初中学生的数学学习 App。

核心目标：
不是简单做题，也不是替学生考试，而是帮助学生真正理解数学、建立完整知识体系，并解决以下问题：

1. 学生经常学过但忘记术语是什么意思。
2. 学生知道公式，但不知道为什么有这个公式。
3. 学生不知道自己的知识漏洞在哪里。
4. 中国数学教材按照章节推进，但知识之间的联系不够明显。
5. 很多数学概念因为术语和符号表达，导致学生感觉困难。

请你帮我设计这个 App 的完整方案。

要求：

一、设计双路线数学体系

路线A：学校同步路线
- 按中国小学、初中数学教材年级和章节组织。
- 让学生可以找到自己正在学校学习的内容。
- 包括章节、小节、知识点。

路线B：理解路线
- 不按照教材顺序，而按照人类理解数学的自然顺序组织。
- 从具体生活经验进入，再逐渐抽象到数学定义。
- 建立知识之间的依赖关系。

例如：
“函数”
不要直接从定义开始，而应该：

生活变化
→ 两个量之间关系
→ 输入输出
→ 表格
→ 图像
→ 函数表达式
→ 正式数学定义

二、设计知识点数据库结构

每个数学知识点必须包含：

1. 名称
2. 年级阶段
3. 教材位置
4. 一句话人话解释
5. 生活中的例子
6. 为什么需要这个概念
7. 正式数学定义
8. 常见错误理解
9. 前置知识
10. 后续知识
11. 相关公式
12. 可视化方式
13. AI讲解提示
14. 练习题类型

三、设计AI老师功能

AI不能只回答答案，而应该像真正老师：

- 根据学生年龄调整语言。
- 先解释概念，再解释术语。
- 不直接丢公式。
- 发现学生错误时分析原因。
- 能回答：
  “这个东西为什么存在？”
  “现实里有什么用？”
  “我忘了怎么办？”
  “我为什么不会？”

四、设计复习系统

需要类似记忆系统：

- 自动记录学生学过的知识。
- 根据遗忘规律提醒复习。
- 发现长期遗忘的知识。
- 建立个人数学知识地图。

例如：

学生点击：
“一元一次方程”

系统显示：

你已经掌握：
✓ 等式概念
✓ 加减运算

你可能薄弱：
⚠ 移项
⚠ 分配律

未来会用到：
→ 二元一次方程
→ 函数

五、设计知识图谱

请设计小学到初中的完整数学知识关系。

要求说明：

- 哪些知识是基础节点。
- 哪些知识依赖其他知识。
- 哪些知识容易断层。
- 哪些知识应该提前建立直觉。

七、设计MVP版本

不要一开始做巨大系统。

请规划：

第一版本：
- 最少功能
- 最小知识范围
- 技术实现方式

第二版本：
- 增加什么

第三版本：
- 如何扩展到完整数学体系

八、重点原则

请始终遵守：

数学不是知识点列表，而是一张知识网络。

不要假设学生理解术语。

所有抽象概念必须先有直觉，再有定义。

目标不是让学生背更多，而是让学生知道：
“这个数学概念为什么会出现，它和其他知识有什么关系。”
